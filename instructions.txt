⭐ HOW TO USE buddhaOS_source (Human Instructions)
This is the version you can put in your repo, your docs, your README, or send to anyone who downloads the .zip.

1. Download the folder
Everything runs locally.

Download the .zip

Unzip it

You’ll see a folder called buddhaOS_source (or whatever you named it)

That folder is the OS.

2. Open the OS
Inside the folder, double‑click:

Code
index.html
Your browser will open buddhaOS immediately.

No install.
No setup.
No terminal.
No dependencies.

3. Explore the desktop
You’ll see:

icons

windows

a taskbar

a boot log

draggable panels

apps

ancestor files

docs

Click anything.
Open everything.
This OS is designed to be discovered, not followed.

4. Open the “Creator Kit”
This is where you build your own pages.

Inside the OS, click the icon labeled:

Creator Kit  
(or whatever you named it)

You’ll find:

a starter template

instructions

examples

editable HTML

This is the “make your own webpage” section.

5. Build your own page
Inside the creator-kit folder (in the actual files), you’ll see:

Code
template.html
style.css
script.js (optional)
To make your own page:

Copy template.html

Rename it to anything (e.g., myproject.html)

Open it in any text editor

Replace the text and images

Save

Open it in your browser

You now have a webpage.

6. Learn from the ancestor files
Inside the folder ancestors/ you’ll find:

whale.html

cosmic.html

abitbizaaare.html

These are the lineage of the system.

You don’t need them to build anything —
but reading them teaches you:

gradients

particles

weird text

early Q@w@rd logic

how the OS evolved

They’re examples, fossils, and inspiration.

7. Modify the OS itself
Everything in buddhaOS is editable.

To change:

Desktop icons
Edit index.html → find the <div class="desktop-icons"> section.

Window behavior
Edit system/ui.js.

Window appearance
Edit system/windows.css.

Boot messages
Edit the <div id="bootlog"> in index.html.

Matrix rain
Edit system/matrix.js.

Taskbar
Edit the <div id="taskbar"> in index.html.

Apps
Add new HTML files to /apps and link them in the desktop icons.

This OS is meant to be remixed, not preserved.

8. Use it as a portfolio, landing page, or creative tool
buddhaOS can become:

a portfolio

a digital zine

a product archive

a landing page

a creative playground

a weird internet room

a teaching tool

a brand OS

Just replace the content with your own.

9. Share your version
Once you’ve edited it:

upload it to GitHub Pages

host it on your own domain

send the .zip to friends

fork it

remix it

mutate it

buddhaOS is designed to be forked.

10. The philosophy (short version)
This OS teaches by:

clicking

exploring

opening windows

reading source

copying templates

editing text

refreshing the browser

It’s a teacher without a hat.
The environment is the instructor.
